84  /**
85  * @brief Open an image using an open GFILE and get it ready for drawing
86  * @details Determine the image format and get ready to decode the first image frame
87  * @return GDISP_IMAGE_ERR_OK (0) on success or an error code.
88  *
89  * @param[in] img The image structure
90  * @param[in] f The open GFILE stream.
91  *
92  * @pre The GFILE must be open for reading.
93  *
94  * @note This determines which decoder to use and then initialises all other fields
95  * in the gdispImage structure.
96  * @note The image background color is set to White.
97  * @note There are three types of return - everything OK, partial success and unrecoverable
98  * failures. For everything OK it returns GDISP_IMAGE_ERR_OK. A partial success can
99  * be distinguished from a unrecoverable failure by testing the GDISP_IMAGE_ERR_UNRECOVERABLE
100  * bit in the error code.
101  * A partial success return code means an image can still be drawn but perhaps with
102  * reduced functionality eg only the first page of a multi-page image.
103  * @note @p gdispImageClose() should be called when finished with the image. This will close
104  * the image and its underlying GFILE file. Note that images opened with partial success
105  * (eg GDISP_IMAGE_ERR_UNSUPPORTED_OK)
106  * still need to be closed when you are finished with them.
107  */
108  gdispImageError gdispImageOpenGFile(gdispImage *img, GFILE *f);

230  /**
231  * @brief Prepare for the next frame/page in the image file.
232  * @return A time in milliseconds to keep displaying the current frame before trying to draw
233  * the next frame. Watch out for the special values TIME_IMMEDIATE and TIME_INFINITE.
234  *
235  * @param[in] img The image structure
236  *
237  * @pre gdispImageOpen() must have returned successfully.
238  *
239  * @note It will return TIME_IMMEDIATE if the first frame/page hasn't been drawn or if the next frame
240  * should be drawn immediately.
241  * @note It will return TIME_INFINITE if another image frame doesn't exist or an error has occurred.
242  * @note Images that support multiple pages (eg TIFF files) will return TIME_IMMEDIATE between pages
243  * and then TIME_INFINITE when there are no more pages.
244  * @note An image that displays a looped animation will never return TIME_INFINITE unless it
245  * gets an error.
246  * @note Calling gdispImageDraw() after getting a TIME_INFINITE will go back to drawing the first
247  * frame/page.
248  */
249  delaytime_t gdispImageNext(gdispImage *img);
